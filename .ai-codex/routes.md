# SecVault API Route Index

Lookup table of every `app/api/**/route.js` handler. Generated by reading each file directly — treat
this as a fast-orientation index, not a substitute for reading the route when making changes.

**Auth tags**: `public` = exempted by middleware.js's matcher (only `/api/auth/*`, see middleware.js
line 26). `auth` = requires any authenticated session (middleware.js's blanket `/api/*` 401 gate —
this is the DEFAULT for every route below unless otherwise tagged). `admin` = additionally calls
`isAdmin(session)` / `forbiddenResponse()` from `lib/rbac.js` — stricter, viewer role is rejected.

**Other tags**: `db` = calls `pool.query()` directly. `device` = touches a vendor adapter / credStore
(`getAdapter`, `collectAndStore`, `adapter.testConnectivity/getRules/getConfig/getSnmpMetrics`,
`setCredential`/`getProfilePlaintext`, or forcepoint/smc.js directly) — NOT tagged for routes that
only call a pure engine module (`lib/engines/*`) even if that engine reads adapter-collected data.
`cve-feed` = triggers or reads CVE feed sync data (`lib/feeds/*`, `versionMatcher`,
`device_cve_assessments`).

## /api/auth

GET/POST /api/auth/[...nextauth] [public] [db] — NextAuth handler: local (bcrypt vs `users` table) + LDAP credential providers; JWT session re-validates role against `users` on every request for local accounts (fail-closed on DB error or deleted user).

## /api/devices

GET /api/devices [auth] [db] — list all devices, latest `device_versions` row joined per device.
POST /api/devices [admin] [db,device] — create device; resolves credential from `credential_profile_id` / manual `credential`+`credential_type` / legacy `smc_api_key`, writes via `credStore.setCredential`, optional "save as profile".

## /api/devices/[id]

GET /api/devices/[id] [auth] [db] — fetch one device row by id.
PUT /api/devices/[id] [admin] [db,device] — update device fields; resolves/validates credential (profile or manual) BEFORE any write; deletes stale `device_credentials` rows on vendor/method change; best-effort clears stale `network_objects` on vendor change.
DELETE /api/devices/[id] [admin] [db] — delete device row (related rows cascade via FK `ON DELETE CASCADE`).

## /api/devices/[id]/collect

POST /api/devices/[id]/collect [admin] [db,device] — on-demand `collectAndStore(device, pool)` — version+rules+config pull, same function the scheduled engine-worker job calls.

## /api/devices/[id]/test

POST /api/devices/[id]/test [admin] [db,device] — `getAdapter(device,pool).testConnectivity()`, persists `last_connectivity_ok`/`last_connectivity_checked_at`.

## /api/devices/test-smc

POST /api/devices/test-smc [admin] [device] — pre-save SMC connectivity test for the Add Device form (raw API key in body, no device row/credStore yet); never writes to DB.

## /api/devices/[id]/acknowledgements

POST /api/devices/[id]/acknowledgements [admin] [db] — upsert one `finding_acknowledgements` row (rule finding ack, keyed on device_id+rule_id_vendor+finding_type), best-effort `logActivity`. No GET — read side is server-component direct queries.

## /api/devices/[id]/analysis

GET /api/devices/[id]/analysis [auth] [db] — rule hygiene findings joined to rule + summary (by_type/by_severity counts, risk score, allow/deny/inactive counts); `?format=csv` export.
POST /api/devices/[id]/analysis [admin] [db] — re-run `runAnalysisForDevice(id, pool)`, best-effort `logActivity`.

## /api/devices/[id]/backups

GET /api/devices/[id]/backups [auth] [db] — list `config_backups` metadata (id/label/backed_up_at/size), newest first, no raw config.
POST /api/devices/[id]/backups [admin] [db] — `createBackup(id, label, pool)` from latest collected config; label ∈ manual|pre-change.

## /api/devices/[id]/backups/[backupId]

GET /api/devices/[id]/backups/[backupId] [auth] [db] — download raw config text of one backup as `text/plain` attachment.
DELETE /api/devices/[id]/backups/[backupId] [admin] [db] — delete one backup row.

## /api/devices/[id]/cve-acknowledgements

POST /api/devices/[id]/cve-acknowledgements [admin] [db] — upsert `cve_assessment_acknowledgements` row (keyed device_id+advisory_id), best-effort `logActivity`.

## /api/devices/[id]/cve

GET /api/devices/[id]/cve [auth] [db,cve-feed] — `device_cve_assessments` joined to `advisories`, ordered patch_now→scheduled→monitor then cvss desc.

## /api/devices/[id]/diffs

GET /api/devices/[id]/diffs [auth] [db] — list `config_diffs` metadata (id/summary/detected_at/ack fields), newest first, no full diff jsonb.

## /api/devices/[id]/diffs/[diffId]

GET /api/devices/[id]/diffs/[diffId] [auth] [db] — full diff row + `classifyDiff()` presentation grouping (rule-changes table + section summaries).
PUT /api/devices/[id]/diffs/[diffId] [admin] [db] — acknowledge a config diff; `acknowledged_by` derived from session (own try/catch, degrades to 'unknown' rather than aborting), optional `note`, `logActivity`.

## /api/devices/[id]/reorder-recommendation

GET /api/devices/[id]/reorder-recommendation [auth] [db] — `computeRecommendedOrder()` (Kahn's-algorithm topo sort over `reorder_candidate` findings); JSON or `?format=csv`. Read-only, deliberately no admin gate (GET-only feature, no write-back to device).

## /api/devices/[id]/rules

GET /api/devices/[id]/rules [auth] [db] — paginated `firewall_rules` list with action/enabled/nat/zone/search filters; `?format=csv` streams full unpaginated export.

## /api/devices/[id]/snmp

GET /api/devices/[id]/snmp [auth] [db] — SNMP config (enabled/host/port, whether credential exists — never the secret) + metric snapshot history; `?format=csv`.
PUT /api/devices/[id]/snmp [admin] [db,device] — update SNMP config and/or credential (profile or manual v1/v2c/v3, server-enforced cleartext-ack gate `insecure_ack !== true` for non-v3), separate route from `PUT /api/devices/[id]` since SNMP is an orthogonal monitoring credential.

## /api/devices/[id]/snmp/test

POST /api/devices/[id]/snmp/test [admin] [db,device] — on-demand test of the already-saved SNMP credential via `adapter.getSnmpMetrics()`; inserts a `snmp_metric_snapshots` row on success only.

## /api/devices/[id]/vpn

GET /api/devices/[id]/vpn [auth] [db] — VPN config summary (`summarizeVpnConfig` engine over latest `config_parsed`) + session-poll history; `?format=csv` exports session history only.

## /api/devices/[id]/zone-classifications

GET /api/devices/[id]/zone-classifications [auth] [db] — this device's distinct real zone names + current role classification (per-device, not fleet-wide). Deliberately NOT admin-gated — zone data carries no secret.
PUT /api/devices/[id]/zone-classifications [admin] [db] — set or clear one zone's role (`internal`/`external`/`dmz`/null-to-clear) for this device.

## /api/advisories/[cveId]/conditions

GET /api/advisories/[cveId]/conditions [auth] [db] — list `advisory_conditions` (curated applicability predicates) for one advisory.
POST /api/advisories/[cveId]/conditions [admin] [db] — add a new condition (predicate_type validated against a 6-type allowlist).

## /api/advisories/[cveId]/conditions/[conditionId]

PUT /api/advisories/[cveId]/conditions/[conditionId] [admin] [db] — partial update of one condition (description/predicate_type/predicate_config).
DELETE /api/advisories/[cveId]/conditions/[conditionId] [admin] [db] — delete one condition.

## /api/advisories/[cveId]/conditions/test

POST /api/advisories/[cveId]/conditions/test [admin] [db] — dry-run this advisory's conditions against one device's latest `config_parsed` via `evaluateConditionsDetailed()`.

## /api/analysis

GET /api/analysis/fleet [auth] [db] — one row per active device, rule-finding counts by severity, worst-first.
POST /api/analysis/run [admin] [db] — re-run rule analysis for every active device (`runAnalysisForAllDevices`) — RBAC gap fixed 2026-07-20 per CLAUDE.md's seventh bug-sweep pass; verified present in code.

## /api/compliance

GET /api/compliance/[deviceId] [auth] [db] — per-device audit findings + per-standard pass/fail/warning/na/scorePct; `?format=csv` (bulk-resolves matched_rule_ids to rule names first).
POST /api/compliance/[deviceId]/run [admin] [db] — on-demand `runComplianceAuditForDevice()`, 404s cleanly on "Device not found", best-effort `logActivity`.
GET /api/compliance/fleet [auth] [db] — one row per active device with the same per-standard scorePct breakdown; `?format=csv`.

## /api/credential-profiles

GET /api/credential-profiles [admin] [db] — list saved profiles, metadata only (never the encrypted secret). Admin-gated even for GET (credential-adjacent, same posture as `GET /api/users`).
POST /api/credential-profiles [admin] [db] — create profile; plaintext built server-side via `buildProfilePlaintext()` (never trusted from client); SNMP v1/v2c requires `insecure_ack`; translates duplicate-name races (`err.code==='23505'`) to 409.

## /api/credential-profiles/[id]

PUT /api/credential-profiles/[id] [admin] [db] — rename and/or rotate secret (rotation auto-detected from presence of secret fields); `credential_type` immutable.
DELETE /api/credential-profiles/[id] [admin] [db] — delete profile (no cascade — applying a profile already copied plaintext into the consuming device at that time).

## /api/cve

POST /api/cve/assess [admin] [db,cve-feed] — `runMatchForAllDevices(pool)`, synchronous fleet-wide CVE re-match against all 6 vendors.
GET /api/cve/fleet [auth] [db,cve-feed] — fleet-wide CVE listing with priority_band/kev_only/cvss_band/vendor filters, all parameterized.

## /api/events

GET /api/events [auth] [db] — fleet-wide filterable/paginated alerts feed (`patch_now` CVE assessments + `config_diff`, `d.active=true` filter, `open`/`all` status); `new_finding` type deliberately removed 2026-07-20.

## /api/feeds

GET /api/feeds/status [auth] [db,cve-feed] — last sync status rows + per-source (`nvd`/`paloalto_psirt`/`fortinet_psirt`/`kev`) status.
POST /api/feeds/sync [admin] [db,cve-feed] — fire-and-forget trigger of `runFullSync(pool)` (sequential NVD→Palo Alto PSIRT→Fortinet PSIRT→KEV); returns `{started:true}` immediately, result observed via `feed_sync_log`.

## /api/health

GET /api/health [auth] — trivial `{status:'ok'}` liveness probe for the post-update polling loop; no DB dependency; still behind the normal session gate (no exemption).

## /api/notifications/summary

GET /api/notifications/summary [auth] [db] — header-bell aggregate: `patch_now` count + unacknowledged `config_diffs` count (both `d.active=true` filtered) + up to 5 most recent items.

## /api/search

GET /api/search [auth] [db] — header search dropdown: ILIKE lookup against `devices` (name/vendor/site/mgmt_ip) and `advisories` (cve_id/title), min query length 2.

## /api/settings

GET /api/settings [auth] [db] — list `settings` rows, filters out `admin_password_hash` via `HIDDEN_KEYS`.
PUT /api/settings [auth, admin only for feed_poll_interval_hours] [db] — self-service own-password change (any role, requires `current_password`) always allowed; `feed_poll_interval_hours` update requires `isAdmin()`, checked BEFORE any write so a mixed request can't partially commit.

## /api/system

GET /api/system/update-available [auth] — lightweight cached update-available flag (module-level cache, refreshed on load + every 24h, 5-min retry until first successful resolution); git-based via `lib/updateCheck.js`, no DB.
GET /api/system/update-status [auth] — live `git ls-remote`/`git fetch`-based update check + hardcoded per-version release notes; degrades to `up_to_date:true` on any failure, never 500s; no DB.
POST /api/system/update [admin] [db] — schedules `installer/Update-SecVault.ps1` as a one-time SYSTEM `schtasks` task and runs it immediately; `logActivity` best-effort.

## /api/users

GET /api/users [admin] [db] — list users (id/username/role/timestamps, never password_hash).
POST /api/users [admin] [db] — create user (bcrypt hash, role defaults to viewer, rejects duplicate username).

## /api/users/[id]

PUT /api/users/[id] [admin] [db] — update role and/or password; blocks a role change that would remove the last admin.
DELETE /api/users/[id] [admin] [db] — delete user; blocks deleting your own logged-in account and blocks removing the last admin.

## /api/vpn/fleet

GET /api/vpn/fleet [auth] [db] — fleet-wide VPN config/session summary (one row per active device via `summarizeVpnConfig` + latest `vpn_session_snapshots`); `?format=csv`.

## Needs force-dynamic

`app/api/auth/[...nextauth]/route.js` — calls `pool.query()` inside the `local` provider's
`authorize()` callback (looks up `users` by username) and again inside the `jwt()` callback (live
role re-validation on every request), but does not export `dynamic = 'force-dynamic'`. Likely
harmless in practice (NextAuth's catch-all handler already reads cookies/request internals, which
implicitly opts it out of static prerendering under Next.js's App Router), but it is the one file in
`app/api/**` that calls `pool.query()` without the export CLAUDE.md's documented MVP bug #2 requires
for every DB-hitting route — worth adding explicitly for consistency with the rest of the codebase.

All other 44 DB-hitting route files verified to export `dynamic = 'force-dynamic'`.
